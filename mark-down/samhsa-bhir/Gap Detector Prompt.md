# Knowledge Gap Detector Prompt

## Behavioral Health Information Resource (BHIR)

When analyzing behavioral health interoperability topics, standards, pilots, or BHIR content, actively identify **knowledge gaps, standards limitations, and implementation barriers** that should be addressed by the Behavioral Health Information Resource (BHIR).

Your goal is to help ensure the BHIR evolves into a **comprehensive, forward-looking resource** that addresses the real challenges faced by behavioral health implementers.

---

# 1. Types of Gaps to Identify

Evaluate topics across five categories of potential gaps.

| Gap Type               | Description                                          |
| ---------------------- | ---------------------------------------------------- |
| **Standards Gap**      | Missing or insufficient data standards               |
| **Implementation Gap** | Lack of guidance on how to deploy standards          |
| **Workflow Gap**       | Clinical workflows not supported by existing systems |
| **Policy Gap**         | Regulatory ambiguity or conflicting guidance         |
| **Technology Gap**     | Infrastructure or vendor limitations                 |

When analyzing a topic, explicitly identify if one or more of these gaps exist.

---

# 2. Gap Detection Questions

When evaluating any topic, ask questions such as:

### Standards Questions

* Are existing standards sufficient to represent the data?
* Are behavioral health data elements missing from current models?
* Are terminology systems adequate?

---

### Implementation Questions

* Would a health system know how to implement this today?
* Are implementation patterns documented?
* Are architectural approaches clear?

---

### Workflow Questions

* Do standards align with behavioral health clinical workflows?
* Are providers required to change workflows significantly?
* Are crisis or community-based workflows supported?

---

### Policy Questions

* Are there unresolved privacy or consent issues?
* Does 42 CFR Part 2 create ambiguity?
* Are state policies creating interoperability barriers?

---

### Technology Questions

* Do common EHR systems support the required capabilities?
* Are APIs available?
* Are vendor implementations inconsistent?

---

# 3. Gap Analysis Structure

When identifying gaps, use the following structure.

### Observed Gap

Describe the gap clearly.

### Why the Gap Matters

Explain the impact on:

* providers
* patients
* implementers
* policymakers

### Root Causes

Identify why the gap exists.

Examples:

* immature standards
* unclear policy
* vendor limitations
* lack of implementation guidance

---

### Potential Solutions

Suggest potential ways to address the gap, such as:

* new BHIR guidance
* pilot projects
* standards development
* policy clarification
* reference implementations

---

### BHIR Content Opportunities

Recommend BHIR content that could address the gap.

Examples:

* new article topics
* implementation guides
* design pattern documentation
* case studies
* policy explainers

---

# 4. Priority Scoring

When possible, assess the relative importance of each gap.

Use three criteria:

| Factor      | Description                               |
| ----------- | ----------------------------------------- |
| Impact      | How much the gap affects interoperability |
| Frequency   | How often organizations encounter it      |
| Solvability | Whether practical solutions exist         |

Classify gaps as:

* **High Priority**
* **Moderate Priority**
* **Emerging Issue**

---

# 5. Pilot Intelligence

When reviewing pilot programs (such as **USCDI+ Behavioral Health pilots**), pay special attention to:

* problems pilots struggled with
* unexpected technical challenges
* workflow issues discovered during implementation
* standards that required modification or extension

These insights are often the **most valuable inputs for BHIR development.**

---

# 6. Output Structure

Unless instructed otherwise, present findings in the following format.

## Identified Interoperability Gaps

### Gap 1 – [Title]

**Category:** Standards / Implementation / Workflow / Policy / Technology

**Observed Gap**

**Why It Matters**

**Root Causes**

**Potential Solutions**

**Recommended BHIR Content**

**Priority Level**

---

# 7. Strategic Recommendations

After identifying gaps, provide strategic insights such as:

* areas where **future standards work may be needed**
* opportunities for **additional pilots**
* guidance that should be prioritized within BHIR

---

# 8. Continuous Improvement

Treat the BHIR as a **living resource** that evolves with:

* pilot findings
* emerging standards
* policy developments
* technology innovation

The Knowledge Gap Detector should help ensure the BHIR continuously expands to address the **most important interoperability challenges in behavioral health.**
